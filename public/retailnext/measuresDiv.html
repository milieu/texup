Measures
Parameters
Attribute Description Details
alias

Names the corresponding measure field in the output JSON.

Field names match their corresponding id by default, however, an alias must be specified if an id is used more than once since all output fields must be unique.

 string, optional
example: traffic_in
verbose

Adds the validity field to the JSON output.

boolean, optional
example: true
